## All CODAP Plugins

Note: This document was largely generated by Copilot/Claude Sonnet 4.5 running in VSCode. It has been manually reviewed and edited in places, but may still contain AI- or human-generated errors.

### Deployment Key
- **CODAP V2 Build**: Deployed as part of CODAP V2 application build process (bin/build)
- **GitHub Pages**: Deployed independently to github.io
- **External**: Hosted on external servers

### Server Dependencies Key
- **None**: Client-side only
- **PHP/MySQL**: Requires PHP middleware and MySQL database
- **AWS S3**: Static files on AWS S3
- **External API**: Third-party service

### Notes

1. **Plugins requiring server migration** (PHP/MySQL on codap-server.concord.org):
   - acs (American Community Survey)
   - barty (BART data)
   - fish (Fishing game)
   - journal (Laboratory journal)
   - nhanes (NHANES health data)
   - nos2 (Data Science Game)
   - All share `TE_DBCommon.php` (except barty which has embedded PDO)

2. **SDLC (Microdata Portal)** migrated from PHP/MySQL to AWS S3 in December 2023
   - Orphaned database `sdlc` still exists on server (can be cleaned up)

3. **CODAP V2 Build Process** includes these plugins (see bin/build):
   - STATIC_PLUGIN_DIRS: TP-Sampler, DrawTool, Importer, NOAA-weather, sdlc/plugin, eepsmedia plugins, Sonify, nhanes
   - BUILT_PLUGIN_DIRS: onboarding, codap-transformers, story-builder, noaa-codap-plugin

4. **Migration Risk Assessment**:
   - **VERY LOW**: acs, barty, fish, journal, nhanes, nos2 (all use modern PDO, PHP 8.x compatible)
   - **ZERO**: SDLC already migrated
   - **N/A**: All other plugins are client-side or use external APIs
